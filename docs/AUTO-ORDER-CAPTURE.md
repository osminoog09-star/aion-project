# Авто-фиксация заказов — без нажатий (план)

**Статус: PLANNED (не реализовано).** Владелец зафиксировал проблему: отмечать
«Подача/Везу/Высадил» руками (или голосом) — двойная работа, ведь заказ он уже
принимает в приложении Bolt. AION должен узнавать фазы заказа **сам**.

## Зачем это вообще

Итог смены (все км — включая холостые до точек, топливо по чекам, доход,
прибыль) считается **без всяких отметок** — это база, она уже работает.
Отметки нужны только для деления км на классы (заказ/подача/порожняк) и
распределения топлива по ним. Ценность реальная, но она не должна стоить
водителю ни одного нажатия.

## Варианты источника (по честности и трудозатратам)

1. **Уведомления Bolt (Android NotificationListenerService)** — рекомендуемый.
   Bolt показывает уведомления при назначении заказа, начале и завершении
   поездки. Слушатель уведомлений (с явного разрешения пользователя) переводит
   их в те же `OrderActivityWindow`, что и ручные отметки, с
   `evidence: "platform_import"` (контракт это уже предусматривает).
   - − Нативный модуль (Expo config plugin / кастомный dev client) → нужна
     пересборка APK (EAS) и решение владельца.
   - − Формат уведомлений Bolt не документирован — нужно снять реальные тексты
     уведомлений с устройства (RU/ET локали) до реализации.
   - + Ноль нажатий; работает в фоне; честные события платформы.

2. **Импорт скриншотов Bolt (уже есть OCR)** — частично.
   В распознанных поездках (`OcrTripRow`) есть только `occurredAt` (одна
   временная точка), нет начала/конца поездки → честно построить окна НЕЛЬЗЯ
   (длительность пришлось бы выдумывать). Если появится экран Bolt со временем
   начала И конца — можно строить окна ретроспективно.

3. **Ручные отметки / голос** — остаются как необязательный запасной способ.
   UI уже помечен «необязательно»; итоги смены от них не зависят.

## Решение

- Классы км — **опциональная надстройка**; база (общие км/топливо/прибыль)
  никогда не требует отметок.
- Автозахват строим на уведомлениях Bolt (вариант 1) после: (а) снятия реальных
  текстов уведомлений с устройства владельца, (б) решения владельца на нативную
  пересборку APK.
- Никакой классификации «по догадке» (остановился → значит посадил) — это
  выдумывание данных, запрещено принципом проекта.

## Зависимости

- Вход: реальные тексты уведомлений Bolt с устройства (владелец).
- Нативный слой: notification listener (Android), разрешение пользователя.
- Потребитель: `orderWindowReducer` / `classifyKilometers` — уже готовы,
  evidence `platform_import` уже в контракте.
