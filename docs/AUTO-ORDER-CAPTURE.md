# Авто-фиксация заказов и сумм — без ввода (план)

**Статус: PLANNED (не реализовано).** Владелец зафиксировал проблему: отмечать
«Подача/Везу/Высадил» руками (или голосом) — двойная работа, ведь заказ он уже
принимает в приложении Bolt. AION должен узнавать **сам**: что пришёл заказ
(например, на 4 €), что он выполнен, — и **сам записывать доход**, без ручного
ввода сумм.

## Зачем это вообще

Итог смены (все км — включая холостые до точек, топливо по чекам, доход,
прибыль) считается **без всяких отметок** — это база, она уже работает.
Отметки нужны только для деления км на классы (заказ/подача/порожняк) и
распределения топлива по ним. Ценность реальная, но она не должна стоить
водителю ни одного нажатия.

## Что именно фиксируем автоматически

1. **Фазы заказа** (назначен → еду за пассажиром → везу → завершён) →
   `OrderActivityWindow` для классов км.
2. **Суммы заказов** (заказ на 4 € выполнен → +4 € в доход смены) →
   income ledger, без ручного/голосового ввода.

## Варианты источника (по честности и трудозатратам)

1. **Уведомления Bolt (Android NotificationListenerService)** — рекомендуемый
   для фаз; возможно, и для сумм. Bolt показывает уведомления при назначении
   заказа, начале и завершении поездки. Слушатель уведомлений (с явного
   разрешения пользователя) переводит их в `OrderActivityWindow` с
   `evidence: "platform_import"` (контракт уже предусматривает). Если в текстах
   уведомлений есть цена заказа — оттуда же берём сумму в доход.
   - − Нативный модуль (Expo config plugin / кастомный dev client) → нужна
     пересборка APK (EAS) и решение владельца.
   - − Формат уведомлений Bolt не документирован — нужно снять реальные тексты
     уведомлений с устройства (RU/ET локали) до реализации. Есть ли в них
     суммы — узнаем только по реальным скриншотам шторки.
   - + Ноль нажатий; работает в фоне; честные события платформы.

2. **Accessibility Service (чтение экрана Bolt)** — если сумм в уведомлениях
   нет. Сервис специальных возможностей (с явного включения пользователем в
   настройках Android) может читать карточку заказа на экране Bolt — там цена
   есть всегда. Самый полный источник, но самый инвазивный и хрупкий к
   обновлениям Bolt; тоже нативный модуль. Рассматривать вторым шагом, только
   если уведомления не содержат сумм.

3. **Импорт скриншотов Bolt (OCR — УЖЕ РАБОТАЕТ) — полуавтомат на сегодня.**
   Один скриншот заработка Bolt после смены → AION распознаёт все поездки с
   суммами и записывает доход. Это уже закрывает «сам вбивать суммы» ценой
   одного скриншота. Для окон км скриншот не годится: в `OcrTripRow` только
   `occurredAt` (одна временная точка), начала/конца поездки нет — длительность
   выдумывать нельзя.

4. **Ручные отметки / голос** — остаются как необязательный запасной способ.
   UI уже помечен «необязательно»; итоги смены от них не зависят.

## Решение

- Классы км — **опциональная надстройка**; база (общие км/топливо/прибыль)
  никогда не требует отметок.
- Автозахват строим на уведомлениях Bolt (вариант 1) после: (а) снятия реальных
  текстов уведомлений с устройства владельца — по ним же решаем, есть ли там
  суммы; (б) решения владельца на нативную пересборку APK. Если сумм в
  уведомлениях нет — оцениваем Accessibility-путь (вариант 2) отдельным
  решением владельца.
- До этого «сам вбивает суммы» уже доступен полуавтоматом: скриншот заработка
  Bolt после смены → OCR-импорт всех заказов (вариант 3).
- Никакой классификации «по догадке» (остановился → значит посадил) — это
  выдумывание данных, запрещено принципом проекта. Суммы — только из реальных
  источников (уведомление/экран/скриншот), никогда «средним чеком».

## Зависимости

- Вход: реальные тексты уведомлений Bolt с устройства (владелец).
- Нативный слой: notification listener (Android), разрешение пользователя.
- Потребитель: `orderWindowReducer` / `classifyKilometers` — уже готовы,
  evidence `platform_import` уже в контракте.
